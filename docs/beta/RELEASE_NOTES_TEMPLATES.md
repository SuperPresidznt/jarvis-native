## Notes for Implementers

**Character Limits:**
- App Store: 4000 characters max
- Google Play: 500 characters max (short description), 4000 characters max (full description)

**Best Practices:**
- Lead with the most exciting feature
- Use active voice ("Track your habits" not "Habits can be tracked")
- Front-load each bullet point with the benefit
- Test readability on mobile screens
- Update version numbers and placeholders before publishing

**Placeholders to Replace:**
- [APP_NAME] - Final app name
- [SPECIFIC_BUG_DESCRIPTION] - Actual bug that was fixed
- [NEW_FEATURE_NAME] - Name of the feature being added
- [FEATURE_NAME] - Specific feature name
- [LOCATION] - Where in the app to find it
- [SIMPLE_ACTION] - The action needed to use it
- [KEY_BENEFIT] - Primary user benefit
- [SUPPORT_EMAIL] - Actual support email address
- [VISUAL_INDICATOR] - UI element that highlights new features
- [Preview items] - Actual upcoming features

**Tone Guidelines:**
- Confident but not arrogant
- Excited but not overhyped
- Professional but approachable
- Focus on "you" (the user) not "we" (the company)

**Common Mistakes to Avoid:**
- Don't list features without explaining benefits
- Don't use technical jargon (API, backend, framework, etc.)
- Don't make promises about future features without disclaimers
- Don't oversell with superlatives (amazing, revolutionary, incredible)
- Don't bury the lead - put the best stuff first
